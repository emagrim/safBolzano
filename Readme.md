# Note

## Struttura cartelle e file

* Di base, le immagini erano nel repository.
* La cosa può essere foriera di sventura. È meglio avere le cose separate.
* Montando un volume su /app/public/data, abbiamo cambiato diversi indirizzi anche nel codice.
* L'idea è di avere le immagini che vengono caricate quando si caricano le pagine in formato webp, nella cartella /app/public/data/lightweight/img (e sottocartelle), e invece le immagini della galleria in /app/public/data/gallery/ .
